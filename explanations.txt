1. The MATLAB & Simulink Environment (The Logic Core)
This environment is designed to validate the engineering logic and simulate the exact control system that would run on the elevator's microcontroller (like an ESP32 or PLC).

init_elevator_params.m: This is the initialization script. It defines the hard limits and thresholds for your system. For example, it establishes what temperature constitutes a fever (e.g., >37.8°C), defines sensor polling rates, sets safety weights, and configures the parameters for the UV-C/Mist sterilization systems.

patent_master_setup.m: This script acts as a synthetic data generator. Because you cannot easily hook up a real thermal camera or radar to Simulink during basic testing, this script generates simulated time-series data for your inputs (Thermal, Lidar/Radar, Cough sensors, Touch panels).

state_machine.slx: This is the actual control algorithm. It continuously reads the data generated by the setup script and uses a defined state machine to transition between operational modes. It determines exactly when the elevator goes from "Normal Operation" to "Hazard Detected" and triggers the "Sterilization/Cleaning" state based on the calculated risk.

2. The Python pygame Simulation (The Visual Demo)
While MATLAB proves the math works, the simulation.py file proves the concept works in a way that is easy to understand visually. It translates the logic into an interactive 2D environment.

Core Mechanics of the Python Demo:

Sensor Emulation:

Lidar Scanner: Casts 2D ray traces to detect if a passenger is inside the cabin.

Thermal Camera: Uses the get_thermal_color function to shift the passenger's color from green (healthy) to red/yellow (feverish) based on their simulated body temperature.

Event Triggers: * Users can manipulate the passenger's position via mouse drag and change their temperature using the up/down arrow keys.

Pressing the Spacebar triggers a "Cough" event, utilizing a particle system to show droplet spread inside the cabin.

Dynamic Risk Engine: The simulation continuously calculates a risk_score (0-100).

If a passenger has a fever, risk climbs.

If a passenger coughs, risk spikes.

If a sick passenger touches the elevator buttons, those specific buttons are flagged as contaminated_buttons, further increasing the risk.

Bio-Hazard Protocol (Actuation): * Once the passenger leaves the cabin, the system evaluates the final risk. If the passenger was sick, coughed, or touched buttons, the system enters cleaning_mode.

Visually, this triggers purple mist particles dropping from the ceiling unit and pulsing UV-C lights on the contaminated buttons. The elevator will not accept new passengers until the clean_timer hits zero and resets the system to IDLE.
